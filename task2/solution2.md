### Thoughts on the last question - recommendations to admin

Firstly, there are lots of 500 errors. I would recommend checking for issues with the application files. The backend error logs might provide more details on the root cause.

Lots of the requests appears to be coming from bots. I think we should consider rate limiting or blocking them if necessary to reduce the load.

I noticed many requests using DELETE or PUT methods, which looks suspicious to me. Ordinary users would not send so many DELETE requests to pages like /about. It would be great to use a WAF to filter these requests or configure allowed methods. I see that some of these DELETE requests returned a 500 or even a 200 status code, indicating the server processed them, so this is definitely worth investigating.
